
# 📄 spec/14_qa_test_cases.md

## QA & Test Case Specification

---

## 0. 문서 목적

이 문서는 다음 질문에 **누구나 동일한 답을 내리게 만드는 것**을 목표로 한다.

* 이 현상은 **버그인가, 의도인가?**
* 지금 상태는 **수정이 필요한가, 그대로 두어야 하는가?**
* 라이브 반영 전에 **통과해야 할 최소 품질 기준은 무엇인가?**

> ❗ 원칙
>
> * QA는 “재미 평가자”가 아니다
> * QA는 **설계 의도의 수호자**다

---

## 1️⃣ QA의 최우선 기준

### 1.1 QA 우선순위 정의

QA는 아래 순서로 문제를 탐지한다.

```text
1. 설계 의도와 다른 플레이 경험
2. 진행 불가 / 공정성 붕괴
3. 크래시 / 치명적 오류
4. 시각/사운드 문제
```

> 크래시는 당연히 중요하지만,
> 이 게임에서 가장 위험한 것은 **체감 붕괴**다.

---

## 2️⃣ 밸런스 QA 기준

### 2.1 밸런스 버그 정의

다음은 **버그로 간주한다**.

```text
- 특정 선택 없이는 진행이 불가능한 구조
- 의도되지 않은 스킬/아이템 조합으로 난이도 붕괴
- 강화 수치가 설명과 다르게 적용됨
```

다음은 **버그가 아니다**.

```text
- 특정 스킬이 강함
- 고수가 쉽게 클리어함
- 특정 챕터가 어렵게 느껴짐
```

---

## 3️⃣ 스킬 시스템 QA

### 3.1 자동 발동 규칙 검증

필수 테스트 항목:

```text
- 스킬 쿨타임 정확성
- 여러 스킬 동시 발동 가능 여부
- 스킬 발동 순서가 결과에 영향을 주지 않는지
```

### 3.2 스킬 슬롯 제한 테스트

```text
- 최대 스킬 5개 초과 불가
- 슬롯이 가득 찼을 때 신규 스킬 카드 미등장
- 강화 전용 카드만 등장하는지
```

---

## 4️⃣ 가챠 & 천장 QA

### 4.1 가챠 결과 검증

```text
- 결과 분포가 설정된 범위를 벗어나지 않는지
- 연속 동일 결과 발생 시 천장 정상 작동
```

### 4.2 천장(Pity) QA

반드시 실패로 처리해야 할 케이스:

```text
- 천장 조건 충족 후 미발동
- 천장 보상 체감이 명확하지 않음
```

허용되는 케이스:

```text
- 천장이 예상보다 빠르게 발동
```

---

## 5️⃣ Wave & Boss QA

### 5.1 Wave 타이밍 검증

```text
- 웨이브 시작/종료 타이밍 정확성
- 중간 보스 / 보너스 보스 웨이브 정확성
```

### 5.2 Boss 설계 검증

```text
- 보스는 “벽”이 아니라 “시험”인가?
- 제한 시간 내 처치 가능 여부
- 부활/광고 사용으로 극복 가능 여부
```

---

## 6️⃣ 성벽 / 실패 / 부활 QA

### 6.1 성벽 피해 처리

```text
- 피해 계산 순서 (실드 → 방어 → HP)
- 특정 적 1종으로 즉사 발생 여부
```

### 6.2 부활 시스템 검증

```text
- 부활 후 즉시 사망하지 않는가?
- 부활 효과가 설명과 일치하는가?
- 연속 부활 시 정상 작동 여부
```

---

## 7️⃣ 광고 시스템 QA

### 7.1 광고 트리거 검증

```text
- 광고 시청 조건 정확성
- 광고 미시청 시 불이익 발생 여부 ❌
```

### 7.2 광고 후 결과 검증

```text
- 광고 후 클리어 가능성 상승
- 광고 후 즉시 실패 발생 ❌
```

---

## 8️⃣ 데이터 & 로그 QA

### 8.1 필수 로그 검증

아래 로그는 반드시 기록되어야 한다.

```text
- wave_start / wave_end
- enemy_killed
- skill_cast
- gacha_pull
- pity_triggered
- revive_used
- game_over
```

> KPI 문서와 1:1 매핑되는 로그만 유지한다.

---

## 9️⃣ 글로벌 / 로케일 QA

### 9.1 로케일 검증

```text
- 영어 / 한국어 / 일본어 / 스페인어 / 중국어
- 모든 UI, 카드, 설명 정상 출력
```

### 9.2 텍스트 길이 QA

```text
- 긴 언어에서도 UI 깨짐 없음
- 줄바꿈 / 축약 정상
```

---

## 🔟 최소 QA 테스트 시나리오

아래 시나리오는 **모든 빌드에서 통과 필수**다.

```text
- 무과금 상태 챕터 클리어
- 스킬 5개 슬롯 풀 상태 플레이
- 천장 직전/직후 가챠
- 부활 연속 사용
- 특정 스킬 미보유 상태 고난이도 도전
- 자동 스킬 동시 발동 상황
```

---

## 11️⃣ QA 금지 규칙

QA가 **개입해서는 안 되는 영역**:

```text
- 고수 플레이 기준 난이도 판단
- 개인 취향에 따른 재미 평가
- 단기 데이터 변동에 따른 밸런스 요구
```

---

## 12️⃣ 이 문서로 보장되는 것

* “이건 버그인가?”에 대한 통일된 판단
* 라이브 사고 가능성 최소화
* 설계 의도 훼손 방지
* QA / 개발 / 운영 간 충돌 제거

---

## ✅ 결론

> QA는 게임을 바꾸는 사람이 아니라
> **게임이 설계대로 흘러가게 지키는 사람**이다.

이 문서는 그 역할을 완전히 수행할 수 있도록 설계되었다.
