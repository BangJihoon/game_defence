# 📄 spec/13_analytics_kpi.md

## Analytics & KPI Specification

---

## 0. 이 문서의 목적

이 문서는 다음 질문에 **숫자로만 답할 수 있게 한다**.

* 지금 게임은 **건강한가?**
* 난이도가 **과한가 / 부족한가?**
* 자동 밸런스가 **개입해야 하는 시점인가?**
* 운영자가 **아무것도 하지 말아야 할 상태인가?**

> ❗ 원칙
>
> * KPI는 **의사결정용**이다
> * “보기 좋은 수치”는 버린다
> * 대응하지 않을 데이터도 명시한다

---

## 1️⃣ KPI 계층 구조 (중요도 기준)

모든 지표는 아래 3단계로 분류한다.

```text
Tier 1: 즉각 대응 판단 지표 (Critical)
Tier 2: 추세 관찰 지표 (Watch)
Tier 3: 참고용 지표 (Reference)
```

---

## 2️⃣ Tier 1 – 즉각 대응 KPI (가장 중요)

> 이 수치가 **임계값을 넘으면 자동 또는 수동 개입이 필요**하다.

---

### 2.1 Chapter Clear Rate (챕터 클리어율)

```text
정의:
- 특정 챕터 진입 대비 클리어 성공 비율
```

**정상 범위**

* 40% ~ 60%

**위험 신호**

* 30% 이하 → 난이도 과도
* 70% 이상 → 난이도 부족

**대응 가능 액션**

* enemy_hp_multiplier 조정
* spawn_rate 조정
* 중간 보스 HP 조정

---

### 2.2 Average Reached Wave (평균 도달 웨이브)

```text
정의:
- Run 종료 시 평균 도달 Wave
```

**정상 범위**

* 의도된 설계 Wave ±2

**위험 신호**

* 평균이 지속적으로 하락
* 특정 Wave에서 급락 (벽 발생)

**의미**

* 스킬/적 설계 문제 가능성 높음

---

### 2.3 Revive & Ad Usage Rate (부활/광고 사용률)

```text
정의:
- 실패 후 부활 또는 광고를 사용한 비율
```

**정상 범위**

* 20% ~ 35%

**위험 신호**

* 40% 이상 → 난이도 과도
* 10% 이하 → 긴장감 부족

**중요 해석**

* 광고 사용률 급증은 **과금 유도 성공이 아니라 실패 신호**

---

### 2.4 Skill Meta Dominance Index (스킬 메타 점유율)

```text
정의:
- 특정 스킬의 선택 비율
```

**정상 기준**

* 단일 스킬 ≤ 40%
* 상위 3개 합 ≤ 70%

**위험 신호**

* 특정 스킬 없으면 클리어 불가 패턴

**대응 원칙**

* 즉각 너프 ❌
* 2~3 챕터 뒤 상성 설계로 해소

---

## 3️⃣ Tier 2 – 추세 관찰 KPI

> 즉각 조치보다는 **방향성 판단**에 사용

---

### 3.1 Session Play Time

```text
정의:
- 한 판 평균 플레이 시간
```

**정상 범위**

* 10 ~ 12분

**의미**

* 짧아짐 → 이탈 / 난이도 문제
* 길어짐 → 피로 / 자동화 지루함

---

### 3.2 Retry Rate (재도전율)

```text
정의:
- Run 종료 후 즉시 재시작 비율
```

**건강한 신호**

* 실패 후 재도전율 높음

**위험 신호**

* 실패 후 이탈 급증

---

### 3.3 Gacha Usage per Run

```text
정의:
- Run 당 평균 뽑기 횟수
```

**정상 범위**

* 20 ~ 30회 이상

**해석**

* 너무 적음 → 성장 정체
* 너무 많음 → 코인 밸런스 붕괴

---

### 3.4 Pity Trigger Rate (천장 발동률)

```text
정의:
- 전체 유저 중 천장을 경험한 비율
```

**이상적 상태**

* 소수만 경험
* 경험 후 이탈률 낮음

**위험 신호**

* 천장 후 이탈 증가
* 천장 없이는 진행 불가 체감

---

## 4️⃣ Tier 3 – 참고용 KPI (직접 개입 ❌)

> **보고는 하되, 흔들리지 말아야 할 데이터**

---

### 4.1 High-Skill User Data

* 상위 1~5% 유저
* 극단적 빌드 / 플레이

📌 **대응 금지**

* 이들의 데이터로 전체 밸런스 조정 ❌

---

### 4.2 Short-Term Fluctuation

* 24시간 이내 급변 데이터

📌 **원칙**

* 최소 48~72시간 관찰 후 판단

---

### 4.3 Community Noise Index

* 일부 유저의 강한 불만
* SNS / 커뮤니티 단기 이슈

📌 **원칙**

* 데이터와 교차 검증 필수

---

## 5️⃣ 자동 밸런스 시스템 연동 기준

### 5.1 자동 개입 허용 항목

```text
AutoAdjustable
- enemy_hp_multiplier
- spawn_rate
- boss_hp
```

### 5.2 자동 개입 금지 항목

```text
ManualOnly
- skill_damage
- skill_cooldown
- gacha_probability
```

---

## 6️⃣ KPI 기반 상태 요약 (운영자용)

```text
GameHealthStatus
- HEALTHY: 모든 Tier1 정상
- WARNING: Tier1 1개 이상 경계
- CRITICAL: Tier1 2개 이상 위험
```

> 운영자는
> **“CRITICAL이 아니면 개입하지 않는다”**
