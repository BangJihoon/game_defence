## 다음 개발 작업 목록 (TODO)

> 이 문서는 현재 프로젝트의 우선순위에 따른 다음 개발 작업 목록을 정의합니다.

---

### 1. 인벤토리 시스템 구체화

*   **아이템 장착 로직 구현**:
    *   인벤토리(`InventoryListView`)의 '장착' 버튼 클릭 시, `PlayerDataManager`에 로직을 구현하여 아이템을 상단의 `EquippedItemsView` 슬롯으로 이동시킵니다.
    *   장착된 아이템은 인벤토리 목록에서 사라지거나 '장착 중'으로 표시되어야 합니다.
    *   장착 해제 로직도 함께 구현합니다.

*   **장비 슬롯 강화 시스템 구현**:
    *   인벤토리의 '주문서' 아이템을 '사용' 시, 특정 장비 슬롯의 `level`을 올리는 로직을 `PlayerDataManager`에 구현합니다.
    *   슬롯의 레벨이 10 단위로 오를 때마다, `maxSockets`가 증가하여 UI에 반영되는지 확인합니다.

### 2. 스킬 시스템 심화

*   **스킬 랭크 업 시 변형(Variant) 적용**:
    *   `PlayerDataManager`의 `rankUpSkill` 메소드에 변형 스킬 적용 로직을 추가합니다.
    *   랭크 업 조건을 만족하면, `SkillDefinition`에 정의된 `variants` 목록을 플레이어에게 보여주는 UI를 구현해야 합니다.
    *   플레이어가 변형 스킬을 선택하면, 해당 스킬의 작동 방식이 변경되도록 `SkillSystem`을 수정합니다.

### 3. 캐릭터 시스템 확장

*   **캐릭터 선택 및 적용 로직**:
    *   캐릭터 화면(`CharacterPage`)에서 '선택됨' 또는 '선택하기' 버튼 로직을 구현합니다.
    *   `PlayerDataManager`에 현재 선택된 캐릭터(`activeCharacterId`)를 저장하는 필드를 추가합니다.
    *   인게임(`OverflowDefenseGame`) 시작 시, `activeCharacterId`에 해당하는 캐릭터의 능력치가 적용되도록 로직을 수정합니다.

### 4. 메뉴 화면 기능 구현

*   **상점(Shop) 화면 구현**:
    *   `ShopPage` UI를 구체화합니다.
    *   '재화로 아이템 구매' (캐릭터 카드, 주문서 등) 기능을 `PlayerDataManager`에 구현하고 UI와 연동합니다.
*   **기타 메뉴 화면**:
    *   현재 플레이스홀더로 남아있는 다른 메뉴들의 기획이 구체화되는 대로 기능을 개발합니다.
